## Authentication

All AKS clusters have two categories of users: service accounts managed by AKS, and normal users.

AKS uses client certificates, bearer tokens, an authenticating proxy, or HTTP basic auth to authenticate API requests through authentication plugins.

Information about [AKS API's](https://learn.microsoft.com/en-us/rest/api/aks/) can be found within the link. [Authentication](https://learn.microsoft.com/en-us/rest/api/apimanagement/apimanagementrest/azure-api-management-rest-api-authentication) and be found with the link.
